Problem 3. The 'Winning' Gene


Time Remaining: 1 hrs, 00 min, 31 sec

Not submitted yet


**Note: The memory limit for this problem is 512MB, twice the default.**

After years of hosting games and watching Bessie get first place over and over, Farmer John has realized that this can't be accidental. Instead, he concludes that Bessie must have winning coded into her DNA so he sets out to find this "winning" gene.

He devises a process to identify possible candidates for this "winning" gene. He takes Bessie's genome, which is a string SS of length NN where 1≤N≤30001≤N≤3000. He picks some pair (K,L)(K,L) where 1≤L≤K≤N1≤L≤K≤N representing that the "winning" gene candidates will have length LL and will be found within a larger KK length substring. To identify the gene, he takes all KK length substrings from SS which we will call a kk-mer. For a given kk-mer, he takes all length LL substrings, identifies the lexicographically minimal substring as a winning gene candidate (choosing the leftmost such substring if there is a tie), and then writes down the 00-indexed position pipi where that substring starts in SS to a set PP.

Since he hasn't picked KK and LL yet, he wants to know how many candidates there will be for every pair of (K,L)(K,L).

For each vv in 1…N1…N, help him determine the number of (K,L)(K,L) pairs with |P|=v|P|=v.

INPUT FORMAT (input arrives from the terminal / stdin):

NN representing the length of the string. SS representing the given string. All characters are guaranteed to be uppercase characters where si∈A−Zsi∈A−Z since bovine genetics are far more advanced than ours.

OUTPUT FORMAT (print output to the terminal / stdout):

For each vv in 1…N1…N, output the number of (K,L)(K,L) pairs with |P|=v|P|=v, with each number on a separate line.

SAMPLE INPUT:

8
AGTCAACG

SAMPLE OUTPUT:

11
10
5
4
2
2
1
1

In this test case, the third line of the output is 5 because we see that there are exactly 5 pairs of KK and LL that allow for three "winning" gene candidates. These candidates are (where pipi is 00-indexed): 

(4,2) -> P = [0,3,4]
(5,3) -> P = [0,3,4]
(6,4) -> P = [0,3,4]
(6,5) -> P = [0,1,3]
(6,6) -> P = [0,1,2]
To see how (4,2) leads to these results, we take all 44-mers 
AGTC
GTCA
TCAA
CAAC
AACG
For each 44-mer, we identify the lexicographically minimal length 2 substring 
AGTC -> AG
GTCA -> CA
TCAA -> AA
CAAC -> AA
AACG -> AA
We take the positions of all these substrings in the original string and add them to a set PP to get P=[0,3,4]P=[0,3,4].

On the other hand, if we focus on the pair (4,1)(4,1), we see that this only leads to 22 total "winning" gene candidates. If we take all 44-mers and identify the lexicographically minimum length 11 substring (using A and A' and A* to distinguish the different As), we get 

AGTC -> A
GTCA' -> A'
TCA'A* -> A'
CA'A*C -> A'
A'A*CG -> A'
While both A' and A* are lexicographically minimal in the last 3 cases, the leftmost substring takes precedence so A' is counted as the only candidate in all of these cases. This means that P=[0,4]P=[0,4].
